Proyecto Redes 2

Práctica para el Segundo Parcial

  1. Montar un servidor Apache:
    Crear un formulario con los siguientes campos: nombre, email y mensaje.
    * El uso de CSS o librerías como Bootstrap o Tailwind es opcional pero valorado.
  2. Montar un servidor Node.js usando HTTP:
    Implementar las operaciones HTTP básicas (POST, PUT, PATCH, DELETE) y responder con los códigos de estado correspondientes (200, 201, 400, 500, etc.).
    * Usar Express reduce la puntuación, ya que facilita el desarrollo.
  3. Conectar el formulario al servidor Node.js:
    El formulario debe enviar los datos al servidor Node.js, que los mostrará en la consola.
  4. [Opcional] Conexión con base de datos:
    El servidor Node.js puede registrar los datos recibidos en una base de datos.
    * Se valorará el uso de Docker para montar la base de datos.
  5. [Opcional] Uso de Git y GitHub:
    Se valorará el uso de repositorios bien organizados con commits claros y manejo de ramas.
  6. [Opcional] Implementación de Docker:
    Se darán puntos extra si Docker se usa en cualquier parte del proyecto.

Informe y Presentación

Deberán entregar un informe detallado explicando el trabajo realizado, acompañado de capturas de pantalla del desarrollo.
Además, el día lunes deberán defender el informe demostrando lo siguiente:

  • Montar el servidor Apache2 (cliente).
  • Montar el servidor Node.js (backend).
  • Realizar la conexión entre cliente y servidor, mostrando el registro de datos.
